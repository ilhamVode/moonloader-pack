# Versioning

## Формат

MAJOR.MINOR.PATCH (например: 1.8.3)

- PATCH — мелкий фикс (UI, текст, кнопки, ошибка скачивания)
- MINOR — новая функция (раздел, зависимость, большой UI-блок)
- MAJOR — ломка совместимости (новый формат manifest)

## Когда что менять

### Менеджер (`manager.version`)

- Только когда меняется `modio_manager.lua`
- Не трогать при изменении скриптов

### Скрипты (`scripts[].version`)

- Только когда меняется конкретный скрипт
- Не трогать при изменении менеджера

### Манифест (`updated_at`)

- Менять при любом изменении `manifest.json`
- Это единственное поле, которое обновляется всегда

## Теги

После merge релиза в `main`:

```bash
git checkout main
git tag v1.8.3
git push origin v1.8.3
```
