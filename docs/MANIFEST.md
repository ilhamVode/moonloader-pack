# Manifest.json

manifest.json — каталог скриптов для Modio Manager.

## Поля

```json
{
  "schema": 1,
  "name": "Название сборки",
  "owner": "Владелец",
  "homepage": "https://...",
  "updated_at": "2026-06-21",
  "notes": "Описание сборки",
  "manager": {
    "file": "modio_manager.lua",
    "version": "1.8.3",
    "url": "https://...",
    "changelog": [...]
  },
  "scripts": [...],
  "news": [...]
}
```

## Контракт

manifest.json — это контракт. Ошибка в нём ломает каталог всем пользователям.

Особенно опасные поля:

- `id` — меняется идентификатор — ломаются ссылки
- `file` — неверное имя — менеджер не найдёт файл
- `url` — битая ссылка — скрипт не установится
- `version` — не совпадает с Lua-файлом — ложный статус outdated

## Ссылки

Все ссылки должны указывать на `raw/refs/heads/main/` — стабильную ветку.

Для тестовой сборки — `raw/refs/heads/dev/`.

## Требования к Lua-файлу

Менеджер определяет версию скрипта, читая первые 8KB (~100-140 строк) файла. Поэтому **обязательные поля должны быть в начале файла**, в первых 10 строках:

```lua
script_name('MyScript')
script_author('YourName')
script_version('1.0')
```

Менеджер также распознаёт форматы:
- `script_version('1.0')` — строка в кавычках
- `script_version(MY_VAR)` — переменная, определённая выше как `MY_VAR = '1.0'`
- `SCRIPT_VERSION = '1.0'` — глобальная переменная
- `SCRIPT_META = { VERSION = '1.0' }` — мета-таблица

Если `script_version` нет в первых 8KB — версия отображается как «без версии».
